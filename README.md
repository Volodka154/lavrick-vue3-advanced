# Учебный проект (Vue 3)

Демонстрационное SPA веб‑приложение интернет‑магазина на **Vue.js 3**. Проект сделан как учебный пример и показывает типичный функционал e‑commerce платформы: просмотр каталога товаров, работа с корзиной, оформление заказа, административная часть и система уведомлений.

> В проекте используется **мокированный API** (симуляция запросов): данные берутся из локальных файлов/заглушек, реальный сервер не требуется, но возможно его подключение.

## Возможности

- **Каталог товаров**: список продуктов, переход на детальную страницу товара
- **Корзина**: добавление/удаление товаров, пересчёт суммы
- **Оформление заказа**: форма завершения покупки
- **Админ‑панель**: управление товарами/заказами (демонстрационный функционал)
- **Уведомления (alerts)**: показ сообщений об ошибках и событиях приложения

## Технологии

- **Vue.js 3 (Composition API)** — построение интерфейса, реактивность, компоненты  
- **Vue Router 4** — маршрутизация в SPA
- **Vuex 4** — централизованное состояние (модули, например: `cart`, `products`, `user`, `alerts`)
- **Axios** — HTTP‑клиент для запросов (в данном проекте — к мок‑слою)
- **Bootstrap 5** — стилизация и готовые UI‑классы/компоненты
- **Vue CLI** — сборка и dev‑сервер
- **Babel** — транспиляция JS для совместимости

## Быстрый старт

### Требования
- **Node.js** (желательно LTS)
- **npm** или **yarn**

### Установка зависимостей
```bash
npm install
# или
yarn
```

### Запуск в режиме разработки
```bash
npm run serve
```

### Сборка для продакшена
```bash
npm run build
```


## Структура проекта (типовая)

- `src/components/` — переиспользуемые компоненты UI
- `src/views/` — страницы (route‑views)
- `src/router/` — маршруты приложения
- `src/store/` — Vuex‑хранилище и модули
- `src/api/` — слой работы с API (в т.ч. мокированные запросы)
- `src/assets/` — статические ресурсы

## Примечания

- Проект задуман как **учебный/демонстрационный**, поэтому мок‑слой API можно заменить реальным сервером без изменения UI‑логики (достаточно адаптировать слой `src/api/`).
